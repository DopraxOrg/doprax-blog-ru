# Руководство по настройке doprax.yaml

Файл `doprax.yaml` служит основным чертежом (blueprint) для вашего приложения на платформе Doprax. Разместив этот файл в **корневом каталоге** вашего проекта, вы сможете автоматизировать настройку окружения, определять архитектуру сервисов и управлять постоянным хранением данных.

---

## 1. Определение сервисов
Раздел `services` — это место, где вы описываете отдельные контейнеры, составляющие ваше приложение.

* **Название сервиса (Service Name):** Уникальный идентификатор для каждого компонента.
* **Образ (Image):** Укажите конкретный Docker-образ (например, `python:3.10` или `nginx:latest`).
* **Контекст сборки (Build Context):** Если вы собираете приложение из исходного кода, укажите `context` (обычно `.`) и путь к `Dockerfile`.
* **Команда (Command):** Основная команда запуска, которая выполняется при старте контейнера (переопределяет стандартную инструкцию CMD в Docker).

## 2. Порты и сеть
Определите, как трафик будет поступать к вашим сервисам.
* **Внутренний порт (Internal Port):** Порт, который ваше приложение прослушивает внутри контейнера.
* **Внешний порт (External Port):** (Опционально) Сопоставление для конкретного публичного доступа.
* **Протокол:** Обычно используются `http` или `tcp`.

## 3. Переменные окружения (`envs`)
Управляйте конфигурацией и настройками безопасности вашего приложения.
* **Статические значения:** Указывайте нечувствительные данные прямо в YAML.
* **Секреты:** Для конфиденциальных данных (например, API-ключей) используйте панель управления Doprax, чтобы определить переменные, которые будут безопасно внедрены во время работы.

## 4. Постоянное хранилище (Volumes)
Стандартное хранилище контейнера является временным. Чтобы данные (базы данных или загрузки пользователей) сохранялись после перезапуска:
* **Путь монтирования (Mount Path):** Укажите, куда том будет подключен внутри контейнера (например, `/var/lib/mysql`).
* **Размер (Size):** Определите объем хранилища (например, `1GB` или `5GB`).

## 5. Управление ресурсами
Оптимизируйте производительность, ограничивая аппаратные ресурсы для каждого сервиса:
* **CPU:** Определите количество ядер или доли (shares) процессора.
* **Память (Memory):** Установите лимит оперативной памяти (например, `512MB` или `2GB`).

## 6. Скрипты жизненного цикла
Автоматизируйте задачи на определенных этапах процесса развертывания:
* **Скрипты сборки (Build Scripts):** Запускаются на этапе создания образа (например, `npm install`).
* **Скрипты запуска (Run Scripts):** Выполняются непосредственно перед стартом основного приложения (например, `python manage.py migrate`).

---

## Пример `doprax.yaml`
```yaml
services:
  web-app:
    image: python:3.9-slim
    build:
      context: .
      dockerfile: Dockerfile
    ports:
      - port: 8000
        protocol: http
    envs:
      - name: DEBUG
        value: "false"
    volumes:
      - name: app-storage
        path: /app/data
        size: 2GB
    scripts:
      run:
        - python manage.py migrate
```
