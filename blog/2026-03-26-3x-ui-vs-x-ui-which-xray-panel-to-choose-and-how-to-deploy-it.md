**3x-ui против x-ui: какой панель XRay выбрать и как развернуть её на Doprax**

Если вы запускаете прокси-инфраструктуру — особенно для многопользовательских сценариев (VPN, обходные сервисы, трафик-менеджмент), — выбор правильной панели управления XRay критически важен. Две самые популярные опции: **x-ui** и **3x-ui**. Обе работают поверх XRay, но отличаются архитектурой, возможностями и удобством эксплуатации.

Вот что нужно знать — без шумихи, только факты:

---

### 🔑 Ключевые различия

| Функция                | x-ui                            | 3x-ui                                      |
|------------------------|----------------------------------|--------------------------------------------|
| **Ядро**               | Go + Vue.js (монолит)           | Go + React (модульная архитектура)        |
| **Многопользовательское управление** | Базовое (до ~50 пользователей) | Полноценное: роли, группы, лимиты по трафику и времени |
| **Поддержка Sing-box** | Нет                             | ✅ Встроена (через совместимый конфигуратор) |
| **Реверс-прокси / TLS** | Через внешние решения (например, Nginx) | Встроенный режим с автоматическим Let’s Encrypt |
| **Порт-форвардинг**    | Ручная настройка                 | Готовые шаблоны и визуальный редактор     |
| **API & интеграции**   | Ограниченный REST API            | Полноценный API + поддержка Marzban-совместимых клиентов |

> 💡 Для операторов, которым важны **лимиты трафика**, **автоматическое обновление сертификатов**, **интеграция с Marzban**, или масштаб до сотен аккаунтов — **3x-ui** становится очевидным выбором.

---

### 🚀 Как развернуть 3x-ui на Doprax за 2 минуты

1. Зайдите в [Doprax Dashboard](https://doprax.com/dashboard)  
2. Нажмите **«Create Server»** → выберите провайдера (Hetzner, Vultr, DigitalOcean и др.) и конфигурацию (достаточно 1 vCPU / 1 GB RAM для старта)  
3. В разделе **«App Market»** найдите **«3x-ui»** — нажмите «Deploy»  
4. Укажите домен (или используйте временный subdomain.doprax.app)  
5. Готово: панель доступна по HTTPS, админ-логин и пароль отправлены в интерфейсе.

> ⚙️ Не нужно настраивать Docker, Nginx, Certbot или firewall. Всё — предустановлено, защищено и готово к работе.

---

### 📌 Почему не x-ui?

x-ui отлично подходит для тестов, личного использования или простых прокси. Но при росте нагрузки он требует ручных доработок: кастомные скрипты для лимитов, отдельный reverse proxy, ручное обновление сертификатов, отсутствие аудита действий. Это замедляет развитие — особенно если вы строите сервис для других.

3x-ui решает это «из коробки», сохраняя open-source прозрачность и полный контроль над инфраструктурой.
